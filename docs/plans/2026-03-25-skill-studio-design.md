# Skill Studio — 设计文档

日期：2026-03-25

## 背景

用户反馈通过 Chat 或 OpenCode 创作/修改 Skill 困难。核心痛点：

1. **跟 AI 协作写 Prompt 的流程太复杂** — Chat/OpenCode 协作写完后，注册到 Skill 的步骤多且不直观
2. **写好之后调试验证困难** — 无法方便地实时测试当前 Prompt，缺乏迭代对比
3. **修改已有 Skill 体验差** — 找到、展开、改、无 diff，不知道改对了没有

## 方案：全新 Skill Studio 页面

路由：`/skill-studio`

### 整体布局

三栏结构：

```
┌──────────┬──────────────────────────┬──────────────────────────┐
│  SKILLS  │   PROMPT EDITOR          │   TEST CONSOLE           │
│  列表     │                          │                          │
│  草稿(n) │   [元信息栏]              │   [选择模型]              │
│          │   名称 / 描述             │                          │
│  列表项  │   [编辑区]               │   对话区（流式）          │
│          │   System Prompt          │                          │
│  已发布  │   [工具栏]               │   [清除] [采纳此版本]     │
│  (只读)  │   AI优化/版本历史/保存   │                          │
└──────────┴──────────────────────────┴──────────────────────────┘
```

- 左栏 ~200px，可折叠
- 中间 ~45% 宽度
- 右栏 ~35% 宽度

### 左栏 — Skill 列表

- `[+ 新建]` 按钮 → 中间编辑器切换为空白新建状态
- 草稿/审核中 → 点击加载到编辑器（可编辑）
- 已发布 → 只读模式打开，顶部显示 `[Fork 并编辑]` 按钮
- 当前打开的 skill 高亮

### 中间栏 — Prompt 编辑器

- 元信息区：名称、描述输入框
- 编辑区：代码编辑器风格的 textarea，monospace 字体
- 版本选择器：`v3 ▾` 下拉，切换历史版本（只读预览 + 可回滚）
- 工具栏按钮：
  - `[✦ AI 优化]` — 右栏切换为 AI 协作模式，AI 加载当前 prompt，用户自然语言描述修改意图，AI 展示 diff，用户点"采纳修改"写入编辑器
  - `[版本历史]` — 同版本选择器
  - `[保存版本]` — 弹出小输入框填写变更说明后保存

### 右栏 — 测试控制台 / AI 协作（双模式）

**测试模式（默认）：**
- 模型选择器
- 对话区（滚动，流式输出）
- 输入框 + 发送
- `[清除对话]` — 重置对话
- `[✓ 采纳此版本]` — 满意时一键保存，变更说明默认"测试通过"
- 测试调用当前编辑器内容，**无需先保存**

**AI 优化模式（点击 AI 优化后激活）：**
- AI 预加载当前 prompt
- 用户自然语言描述修改需求
- AI 回复包含修改后的完整 prompt（diff 高亮）
- `[采纳修改]` → 编辑器内容自动替换
- 切回测试模式时保留各自对话历史

### 关键工作流

**创作新 Skill：**
```
[+ 新建] → 填名称/描述 → 写 Prompt
  → [✦ AI优化] → 右栏协作完善 → [采纳修改]
  → 切回[测试模式] → 验证效果
  → [✓ 采纳此版本] → 自动保存为草稿
  → 左栏显示新条目 → 准备好后 [发布]
```

**修改已有 Skill：**
```
左栏点击 → 编辑器加载最新版本
  → 修改 or AI优化 → 测试验证
  → [保存版本]（历史可回滚）
  → 若已发布：[提交审核]
```

**衍生已发布 Skill：**
```
左栏点击已发布 → [Fork 并编辑] → 创建草稿副本 → 进入编辑流程
```

## 实现说明

- 新建页面路由：`/src/app/(app)/skill-studio/page.tsx`
- 测试对话复用 `/api/proxy` + stream 机制，传入临时 system_prompt 而非 skill_id
- AI 优化调用新增后端接口 or 直接在前端拼接 prompt 让 Claude 处理
- 版本历史 API：`GET /skills/:id/versions`，已存在
- 侧边栏新增导航入口
